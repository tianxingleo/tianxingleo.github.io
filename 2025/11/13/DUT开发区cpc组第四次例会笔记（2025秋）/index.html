前缀和

📚 前缀和算法介绍

什么是前缀和？

前缀和是一种预处理技术，用于快速计算数组（或列表）中子区间（或子数组）的和。

给定一个数组 AAA，它的前缀和数组 PPP 的定义是：

  • P[i]P[i]P[i] 存储的是数组 AAA 中从第一个元素到第 iii 个元素（包括第 iii 个元素）的所有元素的和。

用公式表示就是：

P[i]=A[0]+A[1]+⋯+A[i]P[i] = A[0] + A[1] + \dots + A[i]P[i]=A[0]+A[1]+⋯+A[i]

核心思想：空间换时间

前缀和算法的核心思想是**“空间换时间”**。

  1. 预处理（O(n) 时间）： 花费 O(n)O(n)O(n) 的时间（nnn 是数组长度）计算出整个前缀和数组 PPP。
  2. 查询（O(1) 时间）： 之后，任何子区间 A[i…j]A[i \dots j]A[i…j] 的和都可以在 O(1)O(1)O(1) 的时间复杂度内计算出来。

🔍 前缀和的构建与应用

1. 前缀和数组的构建

假设我们有一个原始数组 AAA。

  • 原始数组 AAA: A[0],A[1],A[2],…,A[n−1]A[0], A[1], A[2], \dots, A[n-1]A[0],A[1],A[2],…,A[n−1]
  • 前缀和数组 PPP: P[0],P[1],P[2],…,P[n−1]P[0], P[1], P[2], \dots, P[n-1]P[0],P[1],P[2],…,P[n−1]

计算关系（递推公式）：

P[i]=P[i−1]+A[i]P[i] = P[i-1] + A[i]P[i]=P[i−1]+A[i]

其中，P[−1]P[-1]P[−1] 视为 0（通常在实现时通过索引偏移或在 PPP 数组开头多加一个 0 来处理）。

💡 具体例子（以 0-based 索引为例）

假设原始数组 A=[3,1,4,1,5]A = [3, 1, 4, 1, 5]A=[3,1,4,1,5]：

索引 i A[i] P[i] (计算过程) P[i] (值)
0 3 P[−1]+A[0]=0+3P[-1] + A[0] = 0 + 3P[−1]+A[0]=0+3 3
1 1 P[0]+A[1]=3+1P[0] + A[1] = 3 + 1P[0]+A[1]=3+1 4
2 4 P[1]+A[2]=4+4P[1] + A[2] = 4 + 4P[1]+A[2]=4+4 8
3 1 P[2]+A[3]=8+1P[2] + A[3] = 8 + 1P[2]+A[3]=8+1 9
4 5 P[3]+A[4]=9+5P[3] + A[4] = 9 + 5P[3]+A[4]=9+5 14

所以，前缀和数组 P=[3,4,8,9,14]P = [3, 4, 8, 9, 14]P=[3,4,8,9,14]。

2. 子数组求和的计算

我们的目标是计算子数组 A[i…j]A[i \dots j]A[i…j] 的和（即从索引 iii 到索引 jjj 的所有元素之和，且 i≤ji \le ji≤j）。

根据前缀和的定义：

  • P[j]=A[0]+⋯+A[i−1]+A[i]+⋯+A[j]P[j] = A[0] + \dots + A[i-1] + A[i] + \dots + A[j]P[j]=A[0]+⋯+A[i−1]+A[i]+⋯+A[j]
  • P[i−1]=A[0]+⋯+A[i−1]P[i-1] = A[0] + \dots + A[i-1]P[i−1]=A[0]+⋯+A[i−1]

因此，子数组 A[i…j]A[i \dots j]A[i…j] 的和为：

Sum(A[i…j])=P[j]−P[i−1]\text{Sum}(A[i \dots j]) = P[j] - P[i-1]Sum(A[i…j])=P[j]−P[i−1]

为了简化边界处理（避免 i=0i=0i=0 时出现 P[−1]P[-1]P[−1]），我们通常构建一个**“带 0”的前缀和数组 SSS**，长度为 n+1n+1n+1，其中 S[0]=0S[0]=0S[0]=0。

  • S[k]S[k]S[k] 存储的是数组 AAA 中前 kkk 个元素的和（即 A[0]A[0]A[0] 到 A[k−1]A[k-1]A[k−1] 的和）。
  • S[k]=A[0]+⋯+A[k−1]S[k] = A[0] + \dots + A[k-1]S[k]=A[0]+⋯+A[k−1]

💡 具体例子（使用带 0 的前缀和数组 SSS）

原始数组 A=[3,1,4,1,5]A = [3, 1, 4, 1, 5]A=[3,1,4,1,5] (长度 n=5n=5n=5)。

索引 k (在 S 中) A 中元素个数 S[k] (值)
0 0 0
1 1 (A[0]A[0]A[0]) 3
2 2 (A[0..1]A[0..1]A[0..1]) 4
3 3 (A[0..2]A[0..2]A[0..2]) 8
4 4 (A[0..3]A[0..3]A[0..3]) 9
5 5 (A[0..4]A[0..4]A[0..4]) 14

带 0 的前缀和数组 S=[0,3,4,8,9,14]S = [0, 3, 4, 8, 9, 14]S=[0,3,4,8,9,14]。

计算子数组 A[2…4]A[2 \dots 4]A[2…4] 的和：

A[2…4]=[4,1,5]A[2 \dots 4] = [4, 1, 5]A[2…4]=[4,1,5]，和为 4+1+5=104 + 1 + 5 = 104+1+5=10。

使用前缀和计算：

Sum(A[2…4])=S[4+1]−S[2]=S[5]−S[2]\text{Sum}(A[2 \dots 4]) = S[4+1] - S[2] = S[5] - S[2]Sum(A[2…4])=S[4+1]−S[2]=S[5]−S[2] Sum=14−4=10\text{Sum} = 14 - 4 = 10Sum=14−4=10

结果正确！

差分算法

📚 差分算法介绍

什么是差分数组？

差分数组 DDD 是针对原始数组 AAA 而构建的一个数组。它的核心定义是：

  • D[i]D[i]D[i] 存储的是原始数组 AAA 中**相邻两个元素的差值**。

用公式表示：

D[i]=A[i]−A[i−1](其中 A[−1] 视作 0)D[i] = A[i] - A[i-1] \quad (\text{其中 } A[-1] \text{ 视作 } 0)D[i]=A[i]−A[i−1](其中 A[−1] 视作 0)

换句话说，差分数组 DDD 是原始数组 AAA 的一阶前向差分。

核心思想：更新转化为点操作

差分算法的核心在于：

  1. 构造关系： 原始数组 AAA 是差分数组 DDD 的前缀和数组。

    由 A[i]=D[i]+A[i−1] 可知，A[i]=D[0]+D[1]+⋯+D[i]\text{由 } A[i] = D[i] + A[i-1] \text{ 可知，} A[i] = D[0] + D[1] + \dots + D[i]由 A[i]=D[i]+A[i−1] 可知，A[i]=D[0]+D[1]+⋯+D[i]

  2. 区间更新： 如果要对原始数组 AAA 的一个子区间 A[i…j]A[i \dots j]A[i…j] 同时加上一个值 CCC，我们只需要对差分数组 DDD 中的两个点进行操作。

  3. 最终求值： 在所有区间更新操作完成后，通过对差分数组 DDD 求前缀和，即可还原出最终更新后的数组 AAA。

这种技术将 O(n)O(n)O(n) 的区间更新操作，优化成了 O(1)O(1)O(1) 的差分数组点操作。

🔍 差分数组的构建与操作

1. 差分数组的构建（预处理）

给定一个原始数组 AAA，我们首先构建它的差分数组 DDD。

💡 具体例子

假设原始数组 A=[3,5,8,2,7]A = [3, 5, 8, 2, 7]A=[3,5,8,2,7] (长度 n=5n=5n=5)：

索引 i A[i] A[i−1] (或 0) D[i]=A[i]−A[i−1]
0 3 A[−1]=0A[-1]=0A[−1]=0 3−0=33 - 0 = 33−0=3
1 5 A[0]=3A[0]=3A[0]=3 5−3=25 - 3 = 25−3=2
2 8 A[1]=5A[1]=5A[1]=5 8−5=38 - 5 = 38−5=3
3 2 A[2]=8A[2]=8A[2]=8 2−8=−62 - 8 = -62−8=−6
4 7 A[3]=2A[3]=2A[3]=2 7−2=57 - 2 = 57−2=5

差分数组 DDD: [3,2,3,−6,5][3, 2, 3, -6, 5][3,2,3,−6,5]

验证（求 DDD 的前缀和）：

  • D[0]=3  ⟹  A[0]=3D[0]=3 \implies A[0]=3D[0]=3⟹A[0]=3
  • D[0]+D[1]=3+2=5  ⟹  A[1]=5D[0]+D[1]=3+2=5 \implies A[1]=5D[0]+D[1]=3+2=5⟹A[1]=5
  • D[0]+D[1]+D[2]=5+3=8  ⟹  A[2]=8D[0]+D[1]+D[2]=5+3=8 \implies A[2]=8D[0]+D[1]+D[2]=5+3=8⟹A[2]=8

    … 验证成功。

2. 区间更新操作（O(1)O(1)O(1) 操作）

目标：对原始数组 AAA 的子区间 A[i…j]A[i \dots j]A[i…j] 的所有元素，都加上一个常数 CCC。

我们只需要对差分数组 DDD 进行两个修改：

  1. 在起点 iii 处加上 CCC： D[i]=D[i]+CD[i] = D[i] + CD[i]=D[i]+C
    • 效果：由于 A[i]A[i]A[i] 是 D[0…i]D[0 \dots i]D[0…i] 的和，此操作会使 A[i],A[i+1],…,A[n−1]A[i], A[i+1], \dots, A[n-1]A[i],A[i+1],…,A[n−1] 全部增加 CCC。
  2. 在终点 j+1j+1j+1 处减去 CCC： D[j+1]=D[j+1]−CD[j+1] = D[j+1] - CD[j+1]=D[j+1]−C
    • 效果：此操作会抵消从 A[j+1]A[j+1]A[j+1] 开始的元素受到的影响。
    • A[j+1]=D[0…j]A[j+1] = D[0 \dots j]A[j+1]=D[0…j] 的和 + (D[j+1]−C)(D[j+1]-C)(D[j+1]−C) 的和。抵消了 CCC 的增长。
    • A[j+2],…A[j+2], \dotsA[j+2],… 也会被抵消。

注意： 如果 jjj 是数组的最后一个索引 (n−1)(n-1)(n−1)，那么 j+1=nj+1=nj+1=n 已经超出了 DDD 的范围。此时，我们只需要进行第 1 步操作，因为不需要抵消后续元素。

💡 具体例子（区间更新）

原始数组 A=[3,5,8,2,7]A = [3, 5, 8, 2, 7]A=[3,5,8,2,7]，差分数组 D=[3,2,3,−6,5]D = [3, 2, 3, -6, 5]D=[3,2,3,−6,5]。

操作： 对 A[2…4]A[2 \dots 4]A[2…4]（即 [8,2,7][8, 2, 7][8,2,7]）的每个元素都加上 C=10C=10C=10。

  • 起始索引 i=2i=2i=2，结束索引 j=4j=4j=4。

  1. 起点操作： D[i]=D[2]D[i] = D[2]D[i]=D[2] 加上 101010。

    D[2]=3+10=13D[2] = 3 + 10 = 13D[2]=3+10=13

  2. 终点操作： j=4j=4j=4 是数组末尾， j+1=5j+1=5j+1=5 超出范围，无需进行 D[j+1]D[j+1]D[j+1] 的操作。

更新后的差分数组 D′D'D′: [3,2,∗∗13∗∗,−6,5][3, 2, **13**, -6, 5][3,2,∗∗13∗∗,−6,5]

3. 还原最终数组（前缀和）

所有区间更新操作完成后，我们对 D′D'D′ 求前缀和，得到最终更新后的数组 A′A'A′。

索引 i D′[i] A′[i]=∑k=0iD′[k]
0 3 3
1 2 3+2=53 + 2 = 53+2=5
2 13 5+13=185 + 13 = 185+13=18
3 -6 18+(−6)=1218 + (-6) = 1218+(−6)=12
4 5 12+5=1712 + 5 = 1712+5=17

最终数组 A′A'A′: [3,5,18,12,17][3, 5, 18, 12, 17][3,5,18,12,17]

与期望结果对比： 原始 A=[3,5,8,2,7]A=[3, 5, 8, 2, 7]A=[3,5,8,2,7]。对 A[2…4]A[2 \dots 4]A[2…4] 加上 10：

  • A[0]A[0]A[0]: 333 (不变)
  • A[1]A[1]A[1]: 555 (不变)
  • A[2]A[2]A[2]: 8+10=188 + 10 = 188+10=18
  • A[3]A[3]A[3]: 2+10=122 + 10 = 122+10=12
  • A[4]A[4]A[4]: 7+10=177 + 10 = 177+10=17

    结果完全吻合！

🌟 总结与应用

差分算法的特点

特点 描述 复杂度
构建 预先计算差分数组 DDD。 O(N)O(N)O(N)
区间更新 对 A[i…j]A[i \dots j]A[i…j] 更新，仅需修改 D[i]D[i]D[i] 和 D[j+1]D[j+1]D[j+1]。 O(1)O(1)O(1)

差分算法的应用场景

  1. 大规模区间更新问题： 这是差分数组最主要的应用场景。例如，有一系列活动，每个活动都会在一段时间内（对应数组的一个区间）对某个资源量（数组元素）产生影响，需要计算所有活动结束后资源的最终状态。
  2. 二维差分： 类似于二维前缀和，二维差分数组可以实现对矩阵中矩形区域的批量更新（加减一个常数），操作复杂度也降为 O(1)O(1)O(1)。
  3. 差分约束： 虽然名字相似，但差分约束系统是图论中的一种算法，用于求解形如 Xi−Xj≤CkX_i - X_j \le C_kXi​−Xj​≤Ck​ 的不等式组，与这里的数组差分算法概念上不同，但都利用了差值关系。

离散化

🧐 什么是离散化算法？

核心思想

当数据量 NNN 不大（例如 10510^5105 级别），但数据的值域 VVV 非常大（例如 10910^9109 级别）时，如果我们需要对数据的值进行统计、排序、或作为数组/数据结构的索引，直接使用原值作为索引会导致内存浪费或计算效率低下。

离散化就是将这些巨大的原始数值，替换成它们在所有值中的相对大小排名。

  • 目的： 将原本稀疏、庞大的值域 [1,109][1, 10^9][1,109] 压缩为连续、紧凑的值域 [1,N][1, N][1,N]（其中 N≤105N \le 10^5N≤105）。

为什么需要离散化？

特性 原始数据值域（[1,109]） 离散化后的值域（[1,N]）
大小 巨大、稀疏 紧凑、小巧
内存/效率 难以作为数组下标，可能需要哈希表或浪费大量内存。 可以直接作为数组下标，操作高效。

适用场景

离散化常用于需要利用数据的值作为下标或进行排名比较的算法中，例如：

  1. 统计/计数： 您提出的问题。
  2. 树状数组/线段树： 当需要在巨大值域上进行区间操作时。
  3. 坐标压缩： 在几何问题中，将巨大坐标压缩到小的离散坐标上。

🎯 解决您的具体问题：统计出现次数

问题： 输入不大于 10510^5105 个不大于 10910^9109 的数，统计每个数的出现次数。

在这个问题中：

  • 数据量 N≤105N \le 10^5N≤105。
  • 值域 V≤109V \le 10^9V≤109。

方案对比

方案 优点 缺点 适用于本题？
1. 离散化 + 数组 统计和查询效率极高 (O(1)O(1)O(1)) 需要额外的排序和映射步骤 是，但不如哈希表简单
2. 哈希表 (std::map/unordered_map) 直接使用原值作为键，代码最简洁 理论上 O(Nlog⁡N)O(N \log N)O(NlogN) (map) 或平均 O(N)O(N)O(N) (unordered_map) 最简单高效的方案
3. 直接用数组 效率 O(1)O(1)O(1) 内存消耗过大，无法创建 10910^9109 大小的数组 不可行

虽然哈希表是最简单直接的方案，但为了讲解离散化，我们先展示如何使用离散化来解决这个问题，然后再对比哈希表。

🌟 离散化算法步骤（用于计数）

使用离散化解决此问题的步骤是：

步骤 1: 存储并去重 (O(N log N))

将所有输入的 NNN 个数存入一个数组 A 中，然后复制到另一个数组 B 中，并对 B 进行排序和去重。

现在，数组 B 包含了所有不同的原始值，且它们是升序排列的。B 的下标就是我们所需的离散化值（排名）。

  • 原值 VVV 对应的离散化值 RRR 满足 B[R−1]=VB[R-1] = VB[R−1]=V。

哈希表

🎯 一、 哈希表的基本原理

哈希表的思想非常简单：通过一个函数，将键（Key）直接映射到数组的下标（索引）上。

1. 组成部分

哈希表主要由三部分构成：

部分 作用 对应到数组
键 (Key) 待存储和查找的唯一标识（可以是数字、字符串、对象等）。 输入
哈希函数 (Hash Function) 将任意大小的键值转换为固定范围内的整数。 运算过程
哈希数组 (Hash Array/Bucket Array) 实际存储值（Value）的数组，哈希函数的输出就是数组的下标。 输出结果

2. 工作流程

  1. 存储/插入：
    • 将待存储的键 key 通过哈希函数 HHH 计算得到一个哈希值 h=H(key)h = H(\text{key})h=H(key)。
    • 将哈希值 hhh 通过取模运算 index=h(modM)\text{index} = h \pmod Mindex=h(modM)（MMM 是哈希数组的长度）映射到数组的合法下标。
    • 将 (key, value) 对存入该下标对应的位置。
  2. 查找：
    • 输入待查找的键 key。
    • 使用同样的哈希函数计算出对应的数组下标 index\text{index}index。
    • 直接访问数组 HashArray[index]\text{HashArray}[\text{index}]HashArray[index] 即可快速取出对应的值 value。

🔑 二、 核心组件：哈希函数 (Hash Function)

哈希函数的质量直接决定了哈希表的性能。一个好的哈希函数需要满足以下特性：

  1. 快速计算： 能够在 O(1)O(1)O(1) 或极快的时间内计算出哈希值。
  2. 均匀分布 (Uniformity)： 能够将不同的键值均匀地分散到哈希数组的各个位置，减少“堆积”现象。
  3. 确定性： 相同的输入键必须总是产生相同的哈希值。

在实际实现中，通常会结合多种方法，如乘法、位操作、除法（取模）等，来构建一个高效的哈希函数。

💥 三、 哈希冲突 (Hash Collision)

什么是冲突？

由于键值空间 VVV（例如 101810^{18}1018 种字符串）远远大于哈希数组空间 MMM（例如 10510^5105 个桶），根据鸽巢原理，必然存在不同的键 K1≠K2K_1 \ne K_2K1​=K2​，但它们经过哈希函数计算后，却映射到了同一个数组下标：

H(K1)(modM)=H(K2)(modM)H(K_1) \pmod M = H(K_2) \pmod MH(K1​)(modM)=H(K2​)(modM)

这种情况就是哈希冲突。哈希冲突是不可避免的，因此处理冲突的方法是哈希表设计的关键。

冲突的常见解决方法

处理冲突主要有两种主流方法：

1. 链式地址法 (Chaining)

这是最常用、实现最简单的方案（C++ 的 std::unordered_map 和 Java 的 HashMap 多数采用此方法）。

  • 原理： 将哈希数组的每一个位置（桶）视为一个链表的头节点。
  • 实现： 当发生冲突时，将新的键值对作为链表的新节点，挂载到该哈希桶对应的链表末尾。
  • 操作：
    • 插入： 计算索引，将新元素插入链表的头部或尾部 (O(1)O(1)O(1))。
    • 查找： 计算索引，然后遍历该链表，直到找到匹配的键 (O(1)O(1)O(1) 平均；最坏 O(N)O(N)O(N))。
  • 特点： 允许哈希数组的负载因子（元素总数 / 桶数）大于 1。

2. 开放寻址法 (Open Addressing)

  • 原理： 当哈希函数计算出的位置已被占用时，探查（Probe）哈希数组中的其他空闲位置来存放新元素。
  • 实现： 不需要额外的链表结构，所有元素都直接存放在哈希数组中。
  • 探查策略：
    • 线性探查 (Linear Probing)： 向后以固定步长（通常是 1）依次寻找下一个空桶。
    • 二次探查 (Quadratic Probing)： 向后以平方步长（12,22,32,…1^2, 2^2, 3^2, \dots12,22,32,…）寻找空桶。
  • 操作： 无论是插入、查找还是删除，都需要沿着探查序列进行。
  • 特点： 数组不能被完全填满，通常要求负载因子小于 0.5 或 0.75。删除操作复杂（需要“懒删除”或标记为 TOMBSTONE\text{TOMBSTONE}TOMBSTONE）。

📈 四、 哈希表的性能和优化

1. 负载因子 (Load Factor)

负载因子 α\alphaα 是衡量哈希表满载程度的关键指标：

α=元素数量 (N)桶的数量 (M)\alpha = \frac{\text{元素数量 (N)}}{\text{桶的数量 (M)}}α=桶的数量 (M)元素数量 (N)​
  • 链式地址法： 当 α\alphaα 增大时，链表变长，查找效率下降。
  • 开放寻址法： 当 α\alphaα 接近 1 时，连续空闲桶越来越少，聚集现象严重，性能急剧下降。

2. 扩容 (Resizing/Rehashing)

当负载因子 α\alphaα 超过一个阈值（例如 0.75）时，为了维持 O(1)O(1)O(1) 的性能，需要进行扩容：

  1. 创建一个两倍于原大小的新哈希数组。
  2. 对原数组中的所有元素重新计算哈希值（Rehashing），并将它们插入到新数组中。
  3. 这个过程的时间复杂度是 O(N)O(N)O(N)，但由于它不经常发生，所以平均时间复杂度仍然是 O(1)O(1)O(1)。

总结：为什么哈希表如此重要？

哈希表是实现字典 (Dictionary) 或映射 (Map) 接口的最佳方式，它在许多领域都有应用：

领域 应用实例
编程语言 C++ 的 std::unordered_map，Python 的 dict，Java 的 HashMap 都是基于哈希表实现的。
数据库 用于索引数据记录，实现快速查找。
缓存 Web 服务器或应用程序中的缓存机制（如 Redis）大量使用哈希表。
算法竞赛 统计元素出现次数、查找重复元素、解决两数之和等问题时，哈希表是首选工具。
