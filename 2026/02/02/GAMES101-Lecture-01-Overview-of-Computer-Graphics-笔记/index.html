其次坐标：Homogenous Coordinates

为了能够用矩阵乘法统一表示平移（Translation），引入了第三个坐标分量：www 坐标。

1. 坐标表示定义

  • 2D 点 (2D point)：表示为 (x,y,1)T(x, y, 1)^T(x,y,1)T
  • 2D 向量 (2D vector)：表示为 (x,y,0)T(x, y, 0)^T(x,y,0)T

注意：www 分量为 111 表示位置（点），为 000 表示方向（向量）。这样设计的一个巧妙之处在于：向量具有平移不变性（加上平移量后，www 依旧为 000）。

2. 平移的矩阵表示 (Matrix representation of translations)

通过 3×33 \times 33×3 矩阵，我们可以将平移操作写成矩阵乘法形式：

(x′y′w′)=(10tx01ty001)⋅(xy1)=(x+txy+ty1)\begin{pmatrix} x' \\ y' \\ w' \end{pmatrix} = \begin{pmatrix} 1 & 0 & t_x \\ 0 & 1 & t_y \\ 0 & 0 & 1 \end{pmatrix} \cdot \begin{pmatrix} x \\ y \\ 1 \end{pmatrix} = \begin{pmatrix} x + t_x \\ y + t_y \\ 1 \end{pmatrix}​x′y′w′​​=​100​010​tx​ty​1​​⋅​xy1​​=​x+tx​y+ty​1​​

计算逻辑：

  • x′=1⋅x+0⋅y+tx⋅1=x+txx' = 1 \cdot x + 0 \cdot y + t_x \cdot 1 = x + t_xx′=1⋅x+0⋅y+tx​⋅1=x+tx​
  • y′=0⋅x+1⋅y+ty⋅1=y+tyy' = 0 \cdot x + 1 \cdot y + t_y \cdot 1 = y + t_yy′=0⋅x+1⋅y+ty​⋅1=y+ty​
  • w′=0⋅x+0⋅y+1⋅1=1w' = 0 \cdot x + 0 \cdot y + 1 \cdot 1 = 1w′=0⋅x+0⋅y+1⋅1=1

为什么这样做？

在传统的 2×22 \times 22×2 矩阵中，平移必须写成加法（P′=MP+TP' = MP + TP′=MP+T）。引入齐次坐标后，平移、旋转、缩放都可以统一写成矩阵乘法的形式。这使得多个变换可以通过**矩阵级联（Matrix Composition）**合并为一个矩阵，极大地提高了计算效率。

Lecture 01 Overview of Computer Graphics：

课程主讲人：闫令琪，清华大学本科，System Professor，研究方向是图形学中的渲染和绘制。

课程助教：刘广哲、石宇辰、邓俊辰，后续会根据课程需求添加更多助教。

课程内容：

  1. 什么是图形学？为什么要学？

    • 图形学的定义：合成和操作视觉信息。
    • 应用领域广泛：
      • 游戏：如《只狼》（全局光照，画面亮度是好画面的标准）、《无主之地三》（卡通风格）。
      • 电影：如《黑客帝国》（特效，如子弹时间）、《阿凡达》（面部和动作捕捉）。
      • 动画：如《疯狂动物城》（毛发渲染）、《冰雪奇缘》（烟雾、粒子、衣服、头发模拟）。
      • 设计：如CAD（Computer-Aided Design），汽车设计、室内设计（宜家网站75%的图是渲染的）。
      • 可视化：将三维扫描、CT、核磁共振等数据转化为视觉信息，如地理数据、医学成像。
      • 虚拟现实（VR）与增强现实（AR）：VR是完全虚拟的，AR是现实与虚拟结合，如微软HoloLens设计摩托车。
      • 数字绘画与图像编辑：如iPad手绘、Adobe Photoshop（曲线、差值、透明度等）。
      • 模拟/仿真：如沙尘暴、房屋破坏、黑洞（光线偏折）。
      • 图形用户接口（GUI）：操作系统界面设计（Mac、Windows、Linux）。
      • 字体（Typography）：字体在放大后仍清晰的原理（点阵与矢量图的区别）。
    • 图形学很难，因为它需要对真实世界的理解（物质、光、材质）和新的计算、显示方法。

  2. 课程具体包括哪些内容？

    • 光栅化（Rasterization）：将三维几何形体显示在屏幕上，是实时图形学（每秒30帧，30 FPS）的主要应用，主要用于游戏。
    • 几何（Geometry）：如何表示光滑曲线、曲面，通过细分得到复杂曲面，形状变化时如何保持拓扑结构。
    • 光线追踪（Ray Tracing）：动画和电影中生成真实感画面的方法，效果好但速度慢，目前有实时光线追踪技术。
    • 动画与模拟（Animation & Simulation）：模拟弹性球落地、布料在风中摆动、沙尘暴、黑洞等物理过程。

  3. 课程不包括哪些内容？

    • 硬件编程：不讲Open GL、DirectX、Vulkan等图形API的具体使用，而是讲解背后的原理，让学生能自行学习API。
    • 三维建模或游戏开发：不教Maya、Max等建模工具，也不教Unity、Unreal等游戏引擎的使用，因为这些是工具层面的应用。
    • 计算机视觉：不涉及图像识别、语义分割、三维重建等需要“猜测”和“理解”图像内容的部分，也不特别提及深度学习在图像生成和分析中的应用，因为这些属于图像层面而非三维模型层面的操作。

图形学、计算机视觉、图像/视频处理、计算摄影学之间的关系：

  • 图形学（Graphics）：从三维模型（Model）生成图像（Image），或在模型层面进行模拟和操作。
  • 计算机视觉（Computer Vision）：从图像（Image）识别、分析、理解三维模型（Model），涉及猜测和推断。
  • 图像/视频处理（Image/Video Processing）：在图像（Image）层面进行操作和转换。
  • 计算摄影学（Computational Photography）：通过不同的摄像和捕捉方法，测量光学属性，拍摄特殊图像。
  • 这些领域边界越来越模糊，相互结合紧密。

Lecture 02 Review of Linear Algebra：

  1. 课程资料与访问方式：每节课后，课件将以PDF形式发布，录播视频也会上传至GAMES网站。两者均可通过课程主页访问。
  2. 推荐阅读材料：课程内容与“虎书”（The Tiger Book）中的某些章节相似，建议学生在课前或课后阅读相关章节，但这并非强制要求。
  3. 课程结构回顾：课程分为四部分：光栅化、几何、光线追踪和模拟/动画。
  4. 图形学基础：本节课主要介绍图形学的基础，特别是线性代数。
  5. 图形学依赖的知识领域：图形学依赖于基础数学（线性代数、微积分、统计）、物理（光学、力学，包括波动光学等）、信号处理、数值分析以及美学。
  6. 向量概念：向量表示方向和长度，平移不改变向量。单位向量长度为1，常用于表示方向。
  7. 向量基本操作：
    • 向量求和：可通过平行四边形法则或三角形法则进行几何理解，代数上直接将坐标相加。
    • 坐标表示：在直角坐标系（笛卡尔坐标系）中，向量可用其在各轴上的分量表示，默认是列向量。
    • 向量长度：在直角坐标系中，向量长度通过勾股定理计算。
  8. 向量点乘（Dot Product）：
    • 定义：a · b = |a| |b| cos(θ)，结果是一个标量（数字）。
    • 用途：
      • 计算两个向量之间的夹角余弦，进而得到夹角。
      • 特别适用于单位向量，此时点乘结果直接是夹角余弦。
      • 计算一个向量在另一个向量上的投影。
      • 判定两个向量方向的接近程度（点乘结果接近1表示方向一致，接近-1表示方向相反，0表示垂直）。
      • 判定方向的前后关系（正数表示大致向前，负数表示大致向后，零表示垂直）。
    • 代数计算：对应元素相乘再相加（例如，二维向量 (xa, ya) · (xb, yb) = xaxb + yayb）。
    • 性质：满足交换律、结合律和分配律。
  9. 向量叉乘（Cross Product）：
    • 定义：a × b 得到一个新的向量，其方向垂直于a和b所在的平面，大小为 |a| |b| sin(θ)。
    • 方向判定：使用右手螺旋定则（从a向b旋转，拇指方向即为叉乘结果的方向）。
    • 性质：
      • 不满足交换律：a × b = - (b × a)。
      • 向量叉乘自身结果为零向量。
      • 满足分配律和结合律。
    • 用途：
      • 建立三维直角坐标系（例如，x轴 × y轴 = z轴）。
      • 判定左右关系（例如，在二维平面上，a × b的z分量为正表示b在a的左侧）。
      • 判定内与外关系（例如，通过判断一个点是否始终在三角形所有边的同一侧来确定它是否在三角形内部）。
  10. 矩阵概念：
    • 定义：按行和列排列的一组数。
    • 矩阵乘法：
      • 要求第一个矩阵的列数等于第二个矩阵的行数。
      • 结果矩阵的每个元素是通过第一个矩阵对应行与第二个矩阵对应列的点乘计算得出。
      • 不满足交换律（A * B ≠ B * A），但满足结合律和分配律。
  11. 矩阵操作：
    • 转置：将矩阵的行和列互换（(AB)T = BT AT）。
    • 单位矩阵（Identity Matrix）：对角线上元素为1，其余为0的方阵，与任何矩阵相乘结果不变（I * A = A）。
    • 逆矩阵（Inverse Matrix）：如果A * A⁻¹ = I，则A⁻¹是A的逆矩阵（(AB)⁻¹ = B⁻¹ A⁻¹）。
  12. 向量与矩阵的结合：
    • 向量点乘可以表示为矩阵乘法：a · b = aᵀb。
    • 向量叉乘可以表示为矩阵乘法：a × b = A* b（其中A* 是a向量的对偶矩阵）。
  13. 后续课程预告：下节课将介绍变换（Transform），包括二维变换和齐次坐标（Homogeneous Coordinate）。
  14. 作业安排：将提供“作业零”框架，旨在帮助学生熟悉编程环境和工具，无实质性内容。

Lecture 03 Transformation

  1. 课程概述与回顾

    • 本次是GAMES101现代计算机图形学的第三节课，主要讲解变换。
    • 回顾上节课内容：向量的定义、基本操作（加、乘），特别是点乘（判断前后）和叉乘（判断左右），以及矩阵在图形学中的应用。
    • 预告作业零即将发布，并提醒注册作业提交系统时避免使用QQ邮箱。

  2. 变换的引入与应用

    • 变换的目的： 描述物体或摄像机的运动（动画，如摄像机移动、机器人跳舞、皮克斯台灯动画），以及成像过程（从三维世界到二维平面的投影）。
    • 变换类型： 提到模型变换（Model Transformation）和视图变换（Viewing Transformation）。
    • 重要性： 变换是后续课程中光栅化成像和投影变换的基础。

  3. 二维空间中的常见变换（及矩阵表示）

    • 缩放（Scale）： 将物体沿X轴和Y轴按比例缩放，可使用对角矩阵 [[Sx, 0], [0, Sy]] 表示。
    • 反射（Reflection/Symmetry）： 沿某一轴进行镜像操作，例如沿Y轴反射为 [[-1, 0], [0, 1]]。
    • 切变（Shear）： 沿某一方向对物体进行拉伸或倾斜，例如水平切变为 [[1, a], [0, 1]]。
    • 旋转（Rotation）： 默认绕原点逆时针旋转，角度为θ时，旋转矩阵为 [[cosθ, -sinθ], [sinθ, cosθ]]。
    • 线性变换： 所有能表示为 X' = M * X 形式的变换（M为矩阵，X为向量）称为线性变换。上述缩放、反射、切变、旋转都属于线性变换。

  4. 齐次坐标（Homogeneous Coordinates）

    • 引入原因： 平移变换无法用简单的矩阵乘法表示（X' = M * X + T，多了一个平移向量T），不属于线性变换。为了统一所有变换的表示形式，引入齐次坐标。
    • 定义：
      • 二维点(x, y)表示为(x, y, 1)。
      • 二维向量(x, y)表示为(x, y, 0)。
    • 齐次坐标的优势：
      • 平移变换可以表示为矩阵乘法：[[1, 0, Tx], [0, 1, Ty], [0, 0, 1]] * [x, y, 1]^T = [x+Tx, y+Ty, 1]^T。
      • 向量的平移不变性：向量末位为0，经过平移矩阵乘法后，其坐标不会改变。
      • 点和向量的加减运算：点-点=向量，点+向量=点，向量+向量=向量。
      • 仿射变换（Affine Transformation）： 所有可以用齐次坐标表示为矩阵乘法形式的变换（包括线性变换和平移）都称为仿射变换。
    • 代价： 引入额外的维度（二维点/向量变为三维表示，2x2矩阵变为3x3矩阵），但其带来的统一性远超代价。
    • 齐次坐标矩阵结构： 对于二维仿射变换，齐次坐标矩阵的最后一行固定为 [0, 0, 1]。

  5. 变换的组合与分解

    • 逆变换： 对应于乘以变换矩阵的逆矩阵，效果是回到变换前的状态。
    • 变换的组合：
      • 复杂的变换可以通过一系列简单变换组合得到。
      • 变换的顺序至关重要，因为矩阵乘法不满足交换律。
      • 对于向量X，如果依次应用变换A1, A2, …, An，则最终表示为 An * ... * A2 * A1 * X，即矩阵从右到左依次作用于向量。
      • 多个变换矩阵可以预先相乘得到一个复合矩阵，例如 M = An * ... * A1，然后 M * X。这意味着一个3x3的矩阵可以表示非常复杂的变换。
    • 变换的分解：
      • 将复杂变换分解为简单变换的组合，有助于理解和实现。
      • 围绕任意点旋转的例子： (1) 将旋转中心C平移到原点 (-C)；(2) 绕原点旋转 (Rα)；(3) 将旋转中心从原点移回C (+C)。矩阵表示为 T(C) * R(α) * T(-C) * X。

  6. 三维空间中的变换（类比二维）

    • 齐次坐标： 三维点(x, y, z)表示为(x, y, z, 1)；三维向量(x, y, z)表示为(x, y, z, 0)。
    • 仿射变换矩阵： 4x4矩阵，最后一行固定为 [0, 0, 0, 1]。
    • 矩阵结构： 左上角3x3表示线性变换（如缩放、旋转），最后一列表示平移量(Tx, Ty, Tz)。
    • 变换顺序： 与二维情况一致，4x4的仿射变换矩阵表示的是先进行线性变换，再进行平移。

Lecture 04 Transformation Cont.

  1. 旋转矩阵的数学性质补充

    • 旋转矩阵是正交矩阵，即其逆矩阵等于其转置矩阵（M−1=MTM^{-1} = M^TM−1=MT）。这一性质在求反向旋转变换时非常有用。

  2. 三维变换的基础

    • 齐次坐标：在3D空间中使用 (x,y,z,w)(x, y, z, w)(x,y,z,w) 表示。w=1w=1w=1 表示点，w=0w=0w=0 表示向量。
    • 仿射变换：使用 4×44 \times 44×4 矩阵表示，涵盖了线性变换（缩放、旋转、切变）和平移。
    • 欧拉角：任何复杂的3D旋转都可以分解为绕 x,y,zx, y, zx,y,z 三个轴旋转的组合（Roll, Pitch, Yaw）。
    • 罗德里格斯旋转公式：用于计算绕任意过原点的轴 n\mathbf{n}n 旋转角度 α\alphaα 的矩阵。

  3. 观测变换（Viewing Transformation）的概念

    • 拍照过程类比为 MVP 变换：
      • Model transformation（模型变换）：摆放模型。
      • View transformation（视图变换）：架设相机。
      • Projection transformation（投影变换）：将3D场景变为2D照片。

  4. 视图变换（View/Camera Transformation）

    • 定义相机：需要位置 (e\mathbf{e}e)、观察方向 (g\mathbf{g}g) 和向上方向 (t\mathbf{t}t)。
    • 标准变换：为了简化计算，通常将相机变换到“标准位置”（原点、看向 −Z-Z−Z 轴、向上为 +Y+Y+Y 轴），场景中所有物体随相机同步变换。

  5. 正交投影（Orthographic Projection）

    • 特点：没有“近大远小”现象，平行线保持平行。
    • 操作过程：将空间中的一个长方体 [l,r]×[b,t]×[f,n][l, r] \times [b, t] \times [f, n][l,r]×[b,t]×[f,n] 平移并缩放至一个“标准立方体”（Canonical Cube，范围在 [−1,1]3[-1, 1]^3[−1,1]3）。

  6. 透视投影（Perspective Projection）

    • 特点：符合人类视觉，具有“近大远小”特征，平行线会相交于一点。
    • 推导思路（核心难点）：
      1. 挤压（Squish）：先将透视投影的视锥体（Frustum）挤压成一个长方体。
      2. 应用正交投影：对挤压后的长方体应用正交投影矩阵。
    • 数学推导关键：利用相似三角形原理确定 xxx 和 yyy 的变化，并利用齐次坐标的性质（点乘以非零常数表示同一点）及“近平面不变、远平面中心点不变”的约束条件推导出最终的变换矩阵。
